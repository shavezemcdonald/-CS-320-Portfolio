# CS-320 Portfolio

## About This Repository

This repository contains the work I completed in CS 320: Software Test, Automation, and QA. It includes the Contact Service project, JUnit test files, and my Project Two Summary and Reflections Report. These projects helped me strengthen my understanding of software testing, writing effective unit tests, validating program requirements, and improving software quality through automation.

## Reflection

### How can I ensure that my code, program, or software is functional and secure?

Working on the Contact Service project taught me that writing the code is only part of the job. I made sure my program was functional by creating JUnit tests for every major requirement, including valid and invalid inputs. Testing different situations helped me find problems before they could become bigger issues. I also learned that secure software starts with validating data and preventing bad information from being accepted. Instead of assuming everything would work, I tested for mistakes on purpose to make sure my program handled them correctly.

### How do I interpret user needs and incorporate them into a program?

This course showed me that understanding the requirements before writing code saves a lot of time later. I focused on building each feature exactly as the customer requested instead of adding unnecessary features. Every class and every JUnit test was written to match the project requirements. If a requirement changed, I updated both the code and the tests so everything stayed consistent. That helped me understand that software should solve the user's problem, not just work from a programmer's point of view.

### How do I approach designing software?

I like to break a project into smaller pieces before writing code. In this class I created the classes first, then added the required methods, and finally wrote JUnit tests to verify everything worked correctly. Testing throughout the development process made it easier to catch mistakes early instead of trying to fix everything at the end. This course helped me realize that planning, organization, and continuous testing are just as important as writing the code itself, and those are habits I plan to carry into future software development projects.
